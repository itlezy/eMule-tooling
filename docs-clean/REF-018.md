## Summary

Three targeted removals of dead code that has no path to execution on a modern
eMule network:

1. **PeerCache opcode handlers** — `OP_PEERCACHE_QUERY/ANSWER/ACK` in
   `ListenSocket.cpp` (PeerCache was a short-lived eMule extension, never
   widely deployed, the server infrastructure is gone)
2. **Win95 detection** — runtime `IsWindows95()` / `GetVersion()` checks that
   short-circuit code paths for Windows 95 — cannot execute on Win10+
3. **Legacy INI preference keys** — `FileBufferSizePref` and `QueueSizePref`
   are old key names that were renamed; the old reads add confusion alongside
   the current-key reads in `Preferences.cpp`

## Items

### REFAC_012 — PeerCache opcode handlers in ListenSocket.cpp

```cpp
// ListenSocket.cpp:1285-1287
case OP_PEERCACHE_QUERY:
case OP_PEERCACHE_ANSWER:
case OP_PEERCACHE_ACK:
    // handler code
```

PeerCache never gained network adoption. These handlers add dead switch arms.
Remove the cases and associated handler functions.

Also check `ClientUDPSocket.cpp` and `BaseClient.cpp` for any corresponding
PeerCache send paths and remove them.

### REFAC_015 — Win95 detection remnants

Locate all `GetVersion()`, `IsWindows95()`, or equivalent `LOBYTE(LOWORD(...))`
checks that gate on Win9x/95 runtime detection. Remove the guards; keep only
the Win10+ code path unconditionally.

Primary locations: `EmuleDlg.cpp`, `ListenSocket.cpp`, potentially
`Preferences.cpp` or `ClientList.cpp`.

### REFAC_016 — Legacy INI key reads in Preferences.cpp

```cpp
// Preferences.cpp:2164
m_uFileBufferSize = ini.GetInt(_T("FileBufferSizePref"), 0);  // old key name
// later:
m_uFileBufferSize = ini.GetInt(_T("FileBufferSize"), DEFAULT_FILEBUFFERSIZE);

// Similar pattern for QueueSizePref / QueueSize
```

The old key (`FileBufferSizePref`, `QueueSizePref`) reads a value that is
immediately overwritten by the current-key read. Remove the dead old-key reads.
If backward migration is needed (read old key only if new key is absent),
implement it explicitly and document it; otherwise just delete the old reads.

Note: as part of FEAT-016 (modern limits), the `FileBufferSize` default also
needs raising to 64 MiB. Coordinate so both changes land in one pass on
`Preferences.cpp`.

## Experimental Reference Implementation

**Status in `stale-v0.72a-experimental-clean`:** Substantially done. Split across several commits:

- **PeerCache opcodes**: Removed from `Opcodes.h` in commit `0c5811d` (WIP: clean dead PeerCache opcode references). `OP_PEERCACHE_QUERY/ANSWER/ACK` defines deleted. Corresponding dead opcodes in `BaseClient.cpp` receive paths also removed. The `OP_REQUESTPREVIEW` and `OP_PREVIEWANSWER` opcodes were also deleted alongside the archive preview retirement.
- **Win95 detection**: Removed in commit `b203a0b` (WIP: remove XP SP2 migration logic) and `8dc4dcd` (WIP: remove legacy XP firewall opener). `GetVersion()`-based gates gone; always-Win10 paths remain.
- **Legacy INI keys**: Removed in commit `bf41753` (WIP: remove legacy preferences compatibility keys). `FileBufferSizePref`, `QueueSizePref`, and ~8 other legacy migration reads cleaned up.
- **Proxy types** (PROXYTYPE_NOPROXY etc.): Deleted from `Opcodes.h`. The full `AsyncProxySocketLayer.cpp/h` and `LastCommonRouteFinder.cpp/h` were removed in `70124f1` (FIX: remove proxy product surface and dead socket layers) and `PPgProxy.cpp/h` deleted as well.

**Porting note:** Proxy removal is a larger change that touches socket layers — port REF-017 first, then tackle legacy-key and PeerCache cleanup, then proxy removal. Opcodes.h changes are a simple delete-and-verify pass.
