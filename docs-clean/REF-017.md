## Summary

Multiple layers of dead code remain from the Win9x/NT4/Win98 era: `#if 0` blocks,
`#ifndef WIN9X` guards, `CCM_SETUNICODEFORMAT` Win98 calls, `MAXCON5WIN9X`,
`_MSC_VER<1400` guards, `deadlake PROXYSUPPORT` comments, and stale WinNT/Win98
code comments. None of this code executes on the Win10+ target; it only adds noise
and inhibits static analysis.

## Items

### WWMOD_001 — Win9x/NT4 `#ifndef` guards in stdafx.h

`srchybrid/stdafx.h` contains `#ifndef WINVER` blocks setting `WINVER=0x0500`
(Win2k) and `_WIN32_WINNT=0x0501` (XP). These predate the project's explicit
`_WIN32_WINNT=0x0A00` (Win10) baseline. Remove the guarded assignments; set
only the Win10 values unconditionally.

### WWMOD_002 — `MAXCON5WIN9X` constant in Opcodes.h

```cpp
// Opcodes.h:108
#define MAXCON5WIN9X 10
```

Win9x-era constant for limiting half-open connections on Windows 9x. No longer
referenced in any non-dead code path. Delete the `#define`.

### WWMOD_003 — `CCM_SETUNICODEFORMAT` Win98 calls (5+ files)

`SendMessage(CCM_SETUNICODEFORMAT, TRUE)` calls appear in:
- `MuleListCtrl.cpp:116`
- `MuleTreeCtrl.cpp` (similar pattern)
- At least 3 other list/tree control init sites

On Win10 all common controls are always Unicode. These calls are harmless but
noise. Remove them.

### WWMOD_004 — Win98/ME icon limit comments

Comments citing "Win98/ME can only load 256 icons" near icon-loading code.
Remove the comments; keep the code if it still applies, otherwise remove the
guard.

### WWMOD_005 — `_MSC_VER<1400` dead guards in stdafx.h

```cpp
// stdafx.h
#if _MSC_VER<1400
  typedef unsigned __int8  uint8;
  ...
#endif
```

MSVC 1400 = VS 2005. Project requires MSVC v143 (VS 2022). These guards are
permanently false. Remove the guarded blocks entirely (see also REF-022 for the
typedef modernization).

### WWMOD_009 — WinNT / Win98 prose comments

Scattered code comments referring to "Windows NT", "Win98", "Windows 9x" as
runtime considerations. Update or remove — they confuse readers about what
platforms are actually supported.

### REFAC_011 — `#if 0` dead code blocks

`EmuleDlg.cpp:344` and at least 2 other locations contain `#if 0 ... #endif`
blocks. These are unambiguously dead. Remove them; if the intent must be
preserved, convert to a comment.

### REFAC_014 — `deadlake PROXYSUPPORT` comments in EMSocket.cpp

20+ occurrences of `// deadlake PROXYSUPPORT` marking code that was conditioned
on a never-shipped proxy feature. The code these comments annotate still runs
(it is not behind an `#ifdef`); the comments are misleading noise. Remove the
`// deadlake PROXYSUPPORT` comment text; leave the code intact.
